Linux入门-命令基础

Created2024-04-04|Updated2024-04-15
|Post Views:

基础内容

  1. 命令基础格式
    command [-option] [parameter]
    command: 命令本身
    -options: [可选] 命令的一些选项，可以通过选项控制命令的行为细节
    parameter: [可选] 命令的参数，多数用于命令的指向目标等
  2. 命令行自动在用户home目录上，也就是—>/home/用户名
  3. ls命令:
    ls [-a -l -h] [Linux路径]
    .开头的文件是Linux系统的隐藏文件/文件夹，只有通过-a选项才能看到这些文件
    -l 以列表的形式展示内容，并展示更多信息
    -h 表示以易于阅读的形式，列出文件大小，如K,M,G(表示KB,MB,GB) 同时必须要搭配-l一起使用
    写法上:可以ls -l -a, ls -la, ls -al
  4. cd [Linux路径]
    cd无选项，只有参数，表示切换到哪个目录下面
    若不写参数，表示回到用户的HOME目录
  5. pwd无选项，无参数，直接打印当前的工作目录路径
  6. 绝对路径: 以根目录为起点，描述路径的一种写法，路径描述以/开头
    相对路径: 以当前目录为起点，描述路径的一种写法，路径描述无需以/开头
    这是对于cd命令来说的
    特殊路径符: .表示当前目录 ..表示上一级目录 ~表示HOME目录
    cd ./Desktop 与 cd Desktop效果一致
    cd ~ 可以直接切换到Home目录
  7. mkdir [-p] Linux路径
    参数必填，相对路径或者绝对路径均可
    -p 表示自动创建不存在的父目录，适用于创建连续多层级的目录
    注意点: 创建文件夹需要修改权限，确保操作均在HOME目录内，不要再HOME外操作，否则涉及到权限问题而无法成功
  8. touch Linux路径
    创建文件，参数必填，相对，绝对，特殊路径符都可以用
    cat Linux路径，直接将内容全部展示出来
    more Linux路径，支持翻页，一页页的查看，按空格翻页，按q退出
  9. cp [-r] 参数1 参数2
    -r 可选，用于复制文件夹使用，表示递归
    参数1，Linux路径，表示被复制的文件或文件夹
    参数2，Linux路径，表示要复制去的地方
  10. mv 参数1 参数2
    参数1 表示Linux路径，表示被移动的文件或文件夹
    参数2 表示Linux路径，表示要移动去的地方，如果目标不存在，则进行改名，确保目标存在

  11. rm [-r -f] 参数1 参数2…
    rm 命令可以用来删除文件，文件夹
    -r 选项用于删除文件夹
    -f 表示force强制删除，不会弹出提示确实信息

    rm 命令支持通配符，用来做模糊匹配
    符号    表示通配符，即匹配任意内容(包含空)
    test表示匹配任何以test开头的内容 test表示匹配任何以test结尾的内容
    test表示匹配任何包含test的内容
    注意：rm 命令在处于root(超级管理员)用户的时候要注意使用，su - root 命令进入

  12. which 要查找的命令
    查看所使用的一系列命令的程序文件在哪里
  13. find 起始路径 -name “被查找文件名”
    find 命令去搜索指定的文件
    注意: 为了确保拥有最大的权限，可以在整个系统中完成搜索，切换到root用户获得管理员权限
    find 命令也可以使用通配符
    find 起始路径 -size +|-n[kMG] 注意这里大小写

  14. grep [-n] 关键字 文件路径
    选项-n 可选，表示在结果中显示匹配的行的行号
    关键字表示过滤的关键字，代用空格或者其它特殊符号，建议用””将关键字包围起来
    文件路径表示要过滤内容的文件路径，可作为内容输入端口

    wc [-c -m -l -w] 文件路径
    -c统计bytes数量 -m统计字符数量 -l统计行数 -w统计单词数量
    文件路径表示被统计的文件，可作为内容输入

    管道符: |
    将管道符左边命令的结果作为右边命令的输入
    eg: cat abc.txt | grep kevin 这里grep最后一个文件路径就填的左边的输出结果

  15. echo 输出的内容
    表示要输出的内容，带有空格或\的特殊符号，建议使用双引号包围
    反引号``包围的内容将被作为命令执行
    重定向符: >和>>
    /> 将左侧命令的结果，覆盖写入到符号右侧指定的文件中
    />> 将左侧命令的结果，追加写入到符号右侧指定的文件中
  16. tail [-f -num] Linux路径 可以查看文件尾部内容
    -f 表示持续跟踪，持续跟踪文件最新更改
    -num 表示查看尾部多少行，不填默认10行
  17. vi 文件路径/vim 文件路径
  18. su - [用户名]
    -符号是可选的，表示是否在切换用户后加载环境变量，建议带上
    切换用户后，使用exit命令退回上一个用户，也可以ctrl+d
    sudo 临时赋予root权限
